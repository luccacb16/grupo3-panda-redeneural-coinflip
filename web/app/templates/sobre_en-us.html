NeuralFlip

Developed by Lucca Couto Barberato, Matheus Bessa, Igor Kenji Ueno, Luiz Otávio Mello, Evandro Gabriel Vianna from Group 3 of the Data Processing and Analysis Group (PANDA) of the Department of Computing at UFSCar.

The NeuralFlip project consists of a neural network that is capable of classifying a 50-character sequence composed of only 0s and 1s as: generated by Human or Machine.

Inspiration

The inspiration came from the excelent (and extremely underrated) by MindingTheData:

MindingTheData - Are You Good at Being Random? | Attempting to Fake Randomness

In his video, he uses a RandomForest to classify the sequences, so we thought, why not use a neural network? =)

Dataset

Just like in the video used as reference, we set up a data collection survey through Google Forms to obtain sequences generated by humans. In total we got 115 participants, which resulted in 488 unique and valid sequences.

In addition, the owner of the MindingTheData channel sent us 85 human generated sequences that he collected in his experiment, resulting in 573 sequences. For the machine (pseudo-random) sequences, we used a Python script, generating 573 more examples, in order to maintain the balance between the classes. In total, the dataset has 1140 unique sequences.

From the sequences, several features were extracted, which serve as the inputs of the neural network. Some of them are: number of 0s, number of transitions between 0 and 1, lempel-ziv complexity. In total, X features were created and Y were selected to be used in the neural network after a feature selection process.

Neural Network

We used the PyTorch library to code the neural network model and it has the following architecture:

  • Fully connected;
  • 35 neurons in the input layer;
  • 3 hidden layers with 64, 32 and 16 neurons, all with Leaky ReLU activation function (a = 0.01);
  • Output layer with 1 neuron, with Sigmoid activation function.

The neural network was trained using the Adam optimization algorithm and the Binary Cross Entropy Loss (BCE Loss) loss function. The training was performed for 16 epochs, with a batch size of 16.

A cross-validation was performed with 5 folds. In each fold, evaluation metrics were monitored and the loss in both training and validation. Furthermore, the training and testing set were saved with each fold. You can access the notebook with neural network training here.
